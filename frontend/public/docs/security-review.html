Findings

High Severity

HIGH — REMEDIATED — SEC-01: API Key Authentication

Location: backend/app/main.py (HTTP middleware), backend/app/api/chat.py (WebSocket upgrade)

Fix applied (Mar 15, 2026): API key authentication via API_KEY env var. HTTP routes validate Authorization: Bearer <key> or ?api_key= query param. WebSocket validates ?api_key= on upgrade (closes 4001 if invalid). /health and /docs/* exempt. Auth disabled when API_KEY is empty (dev mode).

Remaining: Integrate with ON24 SSO (SAML/OIDC) for production. Current API key is a shared secret — suitable for internal/staging use.

Recommendation: Implement JWT or OAuth2 bearer token authentication. Validate tokens on WebSocket upgrade and all API routes. Integrate with ON24 SSO (SAML/OIDC).

HIGH — REMEDIATED — SEC-02: Rate Limiting on All Endpoints

Location: backend/app/main.py (slowapi middleware), backend/app/api/chat.py (per-IP WS limiter)

Fix applied (Mar 15, 2026): slowapi middleware on all REST API routes. Per-IP WebSocket message rate limiter: 20 msg/min default (configurable via RATE_LIMIT_PER_MINUTE). REST API: 100 calls/min per IP. Returns HTTP 429 with clear message when exceeded.

Remaining: Add per-user (not just per-IP) limits when authentication supports individual user identity.

Recommendation: Add per-IP or per-session rate limiting via slowapi or custom middleware. Suggested limits: 20 msg/min per user, 100 REST calls/min per IP.

HIGH — REMEDIATED — SEC-03: CORS Restricted

Location: backend/app/main.py

Fix applied (Mar 15, 2026): CORS origins restricted to localhost:3000 and localhost:3001 (configurable via CORS_ORIGINS). allow_methods restricted to GET, POST, PUT, DELETE (was *). allow_headers restricted to Authorization, Content-Type (was *).

Remaining: Update CORS_ORIGINS to production domain(s) before deployment (e.g., https://nexus.on24.com).

Recommendation: Restrict allow_origins to the frontend domain(s) only. In production: ["https://nexus.on24.com"].

Medium Severity

MEDIUM — SEC-04: Anthropic API Key in Environment Variable

Location: .env.local, backend/app/config.py

API keys stored in .env.local file. While gitignored, the key is passed to Docker containers via env_file and visible in docker inspect.

Recommendation: Use a secrets manager (AWS Secrets Manager, HashiCorp Vault) in production. For Docker: use Docker secrets or mount as read-only file.

MEDIUM — SEC-05: WebSocket Has No Connection Limit

Location: backend/app/api/chat.py

No maximum concurrent WebSocket connections enforced. A malicious client could open thousands of connections to exhaust server resources.

Recommendation: Track active connections with a counter. Reject new connections above threshold (e.g., 5,000). Add per-IP connection limit (e.g., 5).

MEDIUM — SEC-06: Conversation History Stored In-Memory Only

Location: backend/app/agents/orchestrator.py

Conversation history is held in Python process memory. No encryption at rest. If process dumps or debug endpoints exist, history could be exposed.

Recommendation: For sensitive deployments, encrypt conversation history in memory or persist to encrypted storage with TTL-based cleanup.

MEDIUM — SEC-07: Admin Agent Can Execute Write Operations

Location: backend/app/agents/admin_agent.py

Admin agent can create events, add/remove registrants via ON24 API. While a confirmation flow exists, the user identity is not verified — any connected user can confirm.

Recommendation: Tie admin operations to authenticated user with role-based access control (RBAC). Only users with "admin" role should be able to confirm write operations.

MEDIUM — SEC-08: Feedback Endpoint Writes to Disk Without Auth

Location: backend/app/api/chat.py — POST /api/feedback

Feedback is appended to data/improvement-inbox-*.txt without authentication. Could be abused to fill disk or inject misleading training data.

Recommendation: Add rate limiting (5 feedback submissions per session per hour). Validate content length. Consider writing to database instead of flat files.

Low Severity

LOW — SEC-09: Debug Mode May Be Enabled in Production

Location: backend/app/config.py — DEBUG env var

Debug mode is controlled by environment variable with no validation that it's disabled in production.

Recommendation: Default DEBUG=false. Add startup warning if DEBUG is enabled and environment is production.

LOW — SEC-10: No HTTPS Enforcement

Location: Docker Compose / nginx config

Application serves over HTTP. No TLS termination configured in Docker setup.

Recommendation: Add TLS termination at load balancer or nginx. Use HSTS header. Redirect HTTP → HTTPS.

LOW — SEC-11: Uploaded Files Served Without Content-Disposition

Location: File upload serving endpoint

Uploaded files may be served inline rather than as attachments, potentially allowing browser-executed content.

Recommendation: Always serve uploaded files with Content-Disposition: attachment. Scan uploads for malware in production.

LOW — SEC-12: SSL Cert Files Temporarily on Disk

Location: backend/app/db/on24_db.py

SSL certificates for ON24 DB are written to temp files. While cleanup is in finally blocks, a crash could leave certs on disk.

Recommendation: Use tempfile.mkdtemp with restrictive permissions (0o700). Consider in-memory SSL context instead.

Informational

INFO — SEC-13: Prompt Injection Defense Is Pattern-Based

Current defense strips known patterns ("ignore previous instructions", "system:" prefix). Sophisticated prompt injection may bypass pattern matching.

Recommendation: Consider output validation (check that agent responses don't leak system prompts). Add monitoring for unusual agent behavior patterns.

INFO — SEC-14: Knowledge Base Uses Cosine Similarity Without Auth

The knowledge base (637 articles / 2729 chunks) is accessible to all users. Content is ON24 help articles, not sensitive, but future additions should be tenant-scoped.

Recommendation: If sensitive content is added to KB, implement per-tenant article scoping.

INFO — SEC-15: Docker Image Uses Root User

Backend Dockerfile may not specify a non-root user. Container processes running as root have broader attack surface.

Recommendation: Add USER nonroot to Dockerfiles. Use distroless or slim base images.
