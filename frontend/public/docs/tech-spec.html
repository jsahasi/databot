Tenant Isolation (Critical)

All queries use WHERE client_id = ANY($N::bigint[]) with the full sub-client hierarchy array. Root client 10710 has 9 sub-clients. The hierarchy is resolved once at startup via a cycle-safe recursive CTE and cached in-process.

Never accept client_id as input from agents or users. Always sourced from server-side config (ON24_CLIENT_ID env var).

Query Performance Rules

  • Cross-event poll queries: always filter to candidate events by goodafter date first (indexed), then join to event_user_x_answer — never start from the answer table (334M rows → 8s+ timeout)
  • Per-query timeout: 10 seconds
  • Default look-back windows: attendance trends = 12 months; audience companies = 1 month
  • Test event filter: LOWER(description) NOT LIKE '%test%' + HAVING registrant_count >= 6

4. Agent System

Tool-Use Loop

Each agent uses the Anthropic SDK messages.create() with tools parameter — NOT the Claude Agent SDK package. Multi-round loop with max 5 iterations per request.

while rounds < 5:
    response = await client.messages.create(model, tools, messages)
    if response.stop_reason != "tool_use": break
    for tool_call in response.content:
        result = await dispatch_tool(tool_call.name, tool_call.input)
        messages.append(tool_result)
    rounds += 1

Orchestrator Safety

  • History rollback: if a sub-agent throws after tool_use is appended to history, pop the dangling entries before re-raising (prevents "400 tool_use without tool_result" cascade)
  • Overly-broad guard: orchestrator detects "list all / dump all" patterns and responds with focused alternatives rather than routing
  • Prompt injection guard: reject messages containing "ignore previous instructions", "act as", "jailbreak", etc.

6. Knowledge Base

  • Source: 637 ON24 Zendesk help articles chunked into 2,729 text segments
  • Embeddings: OpenAI text-embedding-3-small (1536 dimensions), stored as REAL[] in knowledge_base_articles table
  • Search: numpy cosine similarity over all chunks; top-5 results deduped by article title
  • Ingestion: runs as async background task on app startup; batches 100 texts per OpenAI call
  • No-hallucination rule: concierge answers only from retrieved article content; never extrapolates

7. Calendar API

EndpointDescription
GET /api/calendar?year=&month=All events for month with aggregated KPIs for past events. Filters: >=6 registrants OR future date; excludes "test" events.
GET /api/calendar/event/{id}Single event detail: KPIs + poll/survey/resource counts + AI-ACE content (video_library AUTOGEN_ rows) + key takeaways sections (server-side BeautifulSoup parsing).

Key Takeaways Parsing

ON24 key takeaways HTML is parsed server-side by _parse_kt_sections() in calendar.py. Section headings are identified by font-size: 18px spans with text length < 80 chars (length guard prevents quote content from being misdetected as a heading). Sections map to: summary, takeaways, quote, other.

10. Security

  • Tenant isolation: all queries parameterized with server-side client_id array; never from user input
  • Input validation: message length capped at 4,000 chars; session IDs sanitized (alphanumeric + hyphen)
  • Prompt injection guards in all 4 agent system prompts
  • Admin writes: always require explicit confirmed: true from application layer (not user message content)
  • Generic error messages to clients (no stack traces)
  • Audit logging: every tool call written to agent_audit_logs (fire-and-forget)
